# 短视频推荐系统 — 产品决策文档

> 本文档以产品视角复盘本项目:把算法工作翻译成指标体系、权衡决策与上线方案。
> 技术细节见《项目实验报告》,本文只回答"为什么做、怎么衡量、怎么权衡、怎么上线"。

## 1. 背景与问题定义

短视频 feed 的推荐系统同时服务三方:**用户**(高效获得感兴趣内容)、**创作者**(新内容获得公平的分发机会)、**平台**(时长与留存的可持续增长)。本项目针对其中两个结构性矛盾:

- **兴趣理解的时间尺度矛盾**:用户日均消费数百条视频,系统只看近期行为会把人锁死在"昨天的兴趣"里;但全量历史又无法在毫秒级延迟内计算。
- **分发的马太效应**:85% 的视频缺乏足够互动数据(本数据集中曝光 <30 次的视频占交互的 85%),纯行为驱动的推荐天然偏向热门内容,伤害创作者生态与内容多样性。

## 2. 目标与非目标

**目标**:在不增加线上延迟预算的前提下,(a) 提升排序对用户真实兴趣的预测能力;(b) 显著改善长尾/新内容的分发效率;(c) 建立多目标预估能力,为业务权衡提供抓手。

**非目标**:召回策略重构、实时特征系统、端上重排。

## 3. 指标体系

| 层级 | 指标 | 角色 |
|---|---|---|
| 北极星 | 人均有效观看时长(长期留存的最强代理) | 战略方向 |
| 核心离线 | 点击 GAUC(逐用户排序质量) | 迭代主指标 |
| 护栏 1 | 冷启动 item(曝光<10)的排序 AUC | 创作者生态 |
| 护栏 2 | 推荐列表类目多样性(top-K 去重类目数) | 信息茧房 |
| 护栏 3 | 无偏评估指标(随机流量上的 AUC/GAUC) | 防指标虚高 |
| 副目标 | long_view / like 预估 AUC | 多目标融合素材 |

**为什么 GAUC 而非 AUC**:全局 AUC 会被"区分活跃用户与不活跃用户"的信号污染;GAUC 逐用户衡量"给这个人排得对不对",与用户体验直接对应。

**为什么需要护栏 3**:线下评估数据由旧策略产生,存在曝光偏差——新模型可能只是"更像旧策略"而非"更懂用户"。本项目用随机流量验证:主要收益(语义增强 +1.7pt GAUC)在无偏数据上依然成立,同时发现标准日志的 AUC 虚高约 4pt。**产品含义:凡离线提升必须过无偏/在线验证,否则不可信。**

## 4. 关键权衡决策记录

**决策 1:内容理解 vs 更多行为特征**(资源二选一)
选择内容理解(caption 语义)。依据:按 item 热度分桶,语义增益在纯新 item 上 +1.6pt AUC、热门 item 上仅 +0.5pt——收益恰好落在行为特征永远覆盖不到的地方(新内容没有行为)。行为特征的边际收益递减,内容理解打开的是新的供给侧能力:新视频发布即可被理解和分发,创作者冷启动周期缩短。

**决策 2:长序列怎么用**(算力约束下)
暴力加长注意力窗口在 200 条处收益饱和(GAUC 0.5947→0.5946),但延迟与显存线性上涨——性价比曲线明确拒绝这条路。选择检索式:从全量历史中按细粒度类目检索 50 条相关行为,增量 +0.3pt 且计算量可控。**产品含义:同样的指标目标,选择边际成本低的技术路径。**

**决策 3:多目标怎么定权重**(悬而未决,靠数据说话)
点击、时长、互动三个目标存在张力:点击偏好短平快内容,时长偏好沉浸内容,互动偏好情绪内容。本项目用 MMoE 三头预估 + 融合权重扫描给出实测 tradeoff 曲线(`reports/tradeoff.png`):时长权重 α 从 0 提到 0.4,点击几乎无损(-0.06pt)可白拿 +2.1pt 长观命中——这段是免费午餐;α 继续到 1.0 则需用 1pt 点击换 3.5pt 长观,进入真实取舍区。同时发现时长导向会轻微收窄类目多样性(top-10 去重类目 4.94→4.69),印证多样性护栏的必要性。**权重本质是产品决策而非算法决策,算法的职责是把 Pareto 曲线摆上桌。**

**决策 4:诚实处理负结果**
next-item 预训练无增益(数据量不足且信号同源)。选择原样报告并分析适用条件,而非调参硬凑——迭代文化上,一个被理解的负结果比一个被粉饰的正结果值钱。

**决策 5:精排分数最优 ≠ 列表最优**(来自 Demo 实测)
Demo 第一版直接按精排分取 Top-10,结果用户最大兴趣类目 7/10 霸榜——pointwise 排序的必然结果,茧房问题的现场演示。修复:重排层加类目打散(每类目 ≤2 条)。**产品含义:单条内容的预估精度与整个列表的体验是两个优化对象,重排层(多样性、新鲜度、打散规则)是产品体验的必要组件,不是算法的附属品。**

## 5. 上线方案(假设推向生产)

**阶段 0 离线关卡**:GAUC 提升 ≥0.3pt 且三条护栏不劣化,方可进实验。

**阶段 1 AB 实验**:
- 分流:用户维度随机 5% 实验组 / 5% 对照组(item 维度分流会污染共享的行为数据)。
- 周期:≥2 周,覆盖周末效应;前 3 天只观察不决策(新模型的探索期指标会抖动)。
- 观测:核心=人均有效观看时长、人均点击;护栏=次日留存、负反馈率(hate/减少类似)、新视频(<7 天)分发占比、类目多样性。
- 判停:核心指标显著为负,或任一护栏恶化超阈值 → 自动回滚。

**阶段 2 放量**:5%→20%→50%→全量,每档观察一周;保留 1% 长期反转组(holdback)监测长期效应——短期点击涨、长期留存跌的"甜食效应"只有反转组能看见。

**成本核算**:语义表 560MB 常驻显存、编码为离线一次性成本(4060 上 40 分钟/437 万视频);推理侧只增加一次查表 + 线性投影,延迟增量可忽略。收益/成本比显著优于加宽模型。

## 6. 风险与伦理

- **信息茧房**:更准的兴趣建模可能收窄推荐面。缓解:多样性护栏 + 探索流量;本项目的随机曝光数据正是"强制探索"的产物,证明平台可以用可控探索换取无偏认知(行业参照:抖音 2025 年算法透明化)。
- **内容理解的偏差**:caption 由创作者填写,存在堆砌标签/标题党激励。缓解:语义特征只作为表征增强而非直接排序依据,负反馈信号(is_hate)进入多目标监控。
- **数据边界**:全部使用官方公开脱敏数据集,无真实用户隐私。

## 7. 项目结果速览

排序主指标 GAUC 0.5769 → 0.5976(+2.1pt:语义 +1.7,细粒度长序列 +0.3);冷启动 item AUC +1.6pt;无偏评估确认收益真实;多任务三头(点击/时长/互动)就绪,权衡工具化。
