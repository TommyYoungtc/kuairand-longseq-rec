# 面向短视频场景的长序列大模型化推荐排序系统 — 实验报告

> 数据集:KuaiRand-1K(快手真实推荐日志) · 硬件:单张 RTX 4060 · 周期:约 3 周

## 一、项目概述

本项目在快手公开数据集 KuaiRand-1K 上构建了完整的两级推荐系统(召回 + 精排),核心探索三个方向:**超长期用户序列建模**(SIM 式两阶段检索)、**大规模稀疏特征建模**(长尾 hash 分桶)、**语义/世界知识增强**(caption 句向量注入)。所有实验按时间切分、统一评估协议,结论均有消融支撑。

**主要结果**:排序主指标 GAUC 从纯 ID 短序列基线 0.5769 提升到 0.5976(**+2.1pt**),其中语义增强 +1.7pt、细粒度长序列检索 +0.3pt;语义增益在纯新 item 上达 +1.6pt AUC,呈"越冷越有效"的单调梯度,且在随机曝光的无偏评估上依然成立;MMoE 多任务在不损失主任务的前提下额外获得观看时长与点赞两路预估能力。

## 二、数据与评估协议

KuaiRand-1K:1,000 用户 / 437 万视频 / 1,171 万条真实曝光日志(2022.04.08–05.08),12 种反馈信号。用户序列长度中位数 **8,327**、P90 24,418——公开数据集中极少数能真正支撑"超长期序列"研究的数据。

关键处理决策:

- **候选集裁剪与 hash 分桶**:曝光 ≥30 次的 31,699 个视频进入召回候选集;其余 85% 长尾交互哈希进 20 万个 item 桶(`iid_h`),作者哈希进 5 万桶,避免长尾信息折叠丢失。
- **时间切分**:训练 ≤05.01,验证 05.02–05.04,测试 05.05–05.08。负样本 = 曝光未点击,无随机切分带来的泄漏。
- **历史构建无泄漏**:每条排序样本的用户历史严格截取 `time < t`(同毫秒排除),单元测试覆盖。
- **指标**:召回 Recall/NDCG@K(ground truth 为切分期内点击的新 item);排序 AUC + GAUC(按用户分组加权,工业标准)。

## 三、召回层实验

| 模型(1K,test) | R@50 | R@100 | R@500 |
|---|---|---|---|
| 热门 | 0.0021 | 0.0035 | 0.0159 |
| ItemCF | 0.0047 | 0.0080 | 0.0296 |
| 双塔(logQ 修正 + 最优 checkpoint) | **0.0100** | **0.0164** | **0.0599** |

要点:

1. **logQ 采样偏差修正**是双塔成败的关键:in-batch 负采样系统性打压热门 item,修正后 Pure 数据集 val R@50 从 0.07 → 0.27(近 4 倍)。
2. **最优 epoch checkpoint** 替代末轮保存,Pure test R@50 再 +15%(0.199→0.230)。
3. 候选空间从 7.5k(Pure)扩大到 31.7k(1K)时,双塔对 ItemCF 的相对优势从"接近"变为"翻倍"——学习式稠密检索的价值随候选规模放大。

## 四、排序层:核心 2×2 消融(长序列 × 语义)

| GAUC / AUC(1K,test) | 无语义 | +caption 语义 | 语义增益 |
|---|---|---|---|
| DIN(短序列 200) | 0.5769 / 0.7159 | 0.5943 / 0.7291 | +1.74 / +1.32 pt |
| SIM(+长序列检索) | 0.5775 / 0.7148 | **0.5947 / 0.7299** | +1.72 / +1.51 pt |

### 4.1 长序列建模:三次迭代的问题定位

- **v1(同类目最近 K 条)**:与 DIN 指标逐位相同。诊断:用户日均点击约 280 条,"最近 K 条同类目"几乎全落在短窗口内,两支路信息冗余。
- **v2(不相交检索)**:长支路只取短窗口起点之前的行为,val +0.3pt,test 持平。瓶颈进一步定位到检索粒度与长尾表征质量。
- **v3(二级类目检索键,175 类)**:test GAUC **0.5976**,较一级类目版 +0.29pt、较 DIN+语义 +0.33pt——检索粒度从 66 类细化到 175 类后,远期行为的价值才真正释放。

配合序列长度消融(test GAUC:L=50 → 0.5857,100 → 0.5894,200 → 0.5947,400 → 0.5946),完整结论是:**暴力加长窗口在 200 处饱和,检索式长序列(细粒度键)才能拿到窗口之外的增量**——这正是 SIM/ETA/TWIN 工业路线的动机,本项目用自己的数据复现了这条逻辑链。

### 4.2 语义增强:主要涨点来源

方案:bge-small-zh-v1.5 离线编码 437 万视频 caption(覆盖 85.8%)→ PCA 降至 64 维(保留 54.6% 方差)→ 冻结语义表(fp16,约 560MB 常驻 GPU)→ 可训练线性投影后加入 target 与长短序列每个 token 的表征。

**按 item 热度分桶的语义增益**(SIM±sem,test AUC):

| item 训练期曝光 | +语义 | 无语义 | 增益 |
|---|---|---|---|
| 0(纯新) | 0.7295 | 0.7131 | **+1.64pt** |
| 1–9 | 0.7260 | 0.7120 | **+1.40pt** |
| 10–29 | 0.7204 | 0.7122 | +0.82pt |
| 30–99 | 0.7184 | 0.7148 | +0.36pt |
| 100+ | 0.7218 | 0.7167 | +0.51pt |

增益随 item 冷度单调放大——**协同信号越稀疏,内容语义的边际价值越大**。这是"预训练语言模型的世界知识补足推荐系统协同信号"的直接实证,也是行为与语义统一表征(LRM 方向)的动机所在。

## 五、多任务:MMoE

SIM+语义底座 → 4 共享专家 → click / long_view / like 三任务独立门控与塔(1K,test):

| 任务 | AUC | GAUC |
|---|---|---|
| is_click(主) | 0.7302 | 0.5939 |
| long_view | 0.7419 | 0.6152 |
| is_like | 0.9180 | 0.5961 |

主任务与单任务版打平(0.5939 vs 0.5947,差异在噪声内),额外获得两路预估——多目标融合排序的基础。MMoE 的软路由天然可引申至 sparse MoE(Top-k 门控)以承接更大规模下的用户异质性。

## 六、无偏评估与 Case Study

**无偏评估**(test 期随机曝光,4.3 万条等概率展示):SIM+语义 AUC 0.688 / GAUC 0.586,无语义 0.676 / 0.569。两个结论:标准日志的 AUC 虚高约 4pt(曝光偏差被量化);语义增益在无偏数据上依然成立(+1.2pt AUC / +1.7pt GAUC),排除了"增益来自旧策略偏差"的可能。此评估依赖 KuaiRand 独有的随机干预数据,也是 off-policy evaluation / RL 方向的天然入口。

**Case Study**(`reports/case_study.md`):语义近邻示例中同系列视频 cos 0.94、萌娃/追剧/美妆各自成簇;"语义救回"案例——用户近期点击娱乐综艺内容,目标视频《78岁老奶奶身材完爆杨幂…#达人秀》被无语义模型排在 11% 分位、语义模型排到 93% 分位且实际被点击,直观展示世界知识如何补足协同信号。

## 七、生成式预训练(负结果,含分析)

用 2 层 Transformer 在点击流上做自回归 next-item 预训练(in-batch softmax + logQ),将 item embedding 迁移到排序模型微调:test GAUC 0.5945,与随机初始化(0.5947)持平。原因分析:预训练语料仅 64 万事件而词表 23 万,严重欠拟合(loss 停在 4.8);且预训练信号与下游 CTR 同源(都来自点击流),信息增量有限。可推断的适用条件:更大规模、跨域/异源信号(如搜索日志、多场景行为)时预训练才有空间——这与工业界"预训练在推荐中价值依赖数据异质性"的经验一致。

## 八、过程中的方法论发现

- **One-epoch 现象**:CTR 类模型(DIN/SIM/MMoE)在两个数据集上 val GAUC 均在第 1 个 epoch 见顶,复现了工业界大规模稀疏模型"只训一遍"的经典结论;召回双塔则需 8+ epoch 收敛。两类模型训练动力学的差异被早停机制统一处理。
- **评估协议的坑**:随机切分会泄漏、末轮保存损失 15%、in-batch 负采样偏差让双塔看似"不 work"——三个问题都曾在本项目真实出现并被逐一修复。

## 九、局限与后续方向

- **检索继续升级**:硬检索已到二级类目;下一步可用已有语义向量做 embedding 软检索(ETA/TWIN 路线),或三级类目。
- **规模化演进**:统一 token 化已就位(item/作者/类目/语义共入表征空间)。放大即:更大 embedding 表 + Transformer 编码全序列 + sparse MoE 分路 + 多域异源预训练(本项目负结果指明的方向),对应 HSTU/OneRec 等生成式推荐路线;生成式检索可引申语义 ID(TIGER)。
- **数据局限**:1,000 用户使 GAUC 方差偏大;like 等稀疏信号建议引入任务加权或 PLE 进一步解耦。
